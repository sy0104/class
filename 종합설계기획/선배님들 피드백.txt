# 개발 초기에 서버 개발자는 무슨 일을 하나요? 서버를 하는데 다이렉트 지식이 많이 필요한가요?	
	넷겜플만 듣고 구축해도 되나요? 어떤 방식으로 공부했나요?
개발 초기에 해야 할건 클라랑 상관없이 소통을 하는 서버를 만들어 봐야한다.
그 다음 중요한건 우리 게임의 패킷을 정하는 것이 중요하다 (사이즈, 어떤 요소가 중요할지)
나중에 패킷 뜯어고치면 서버 구조 다 바꿔야 할 수 있다.
다이렉트 지식 하나도 없어도 서버 할 수 있다. 대신 수학이랑 물리 지식은 필요 (서버에서 계산함)
넷겜플만 들어도 가능하다 (TCP/IP나 UDP사용)
IOCP에서 통신하는 부분까지 배우면 서버 혼자 만들수있음

# 팀에서 클라이언트를 담당했는데 서버를 어느정도 알아야 할까요?
서버 구조나 어떻게 동작하는지 이해는 해야 한다. 팀원이랑 소통할 때 도움은 되지만 클라이언트라면
서버의 구조는 알 필요 없고 서버가 뭘 하는지는 알아야 한다. 
어떤 데이터는 파일로 읽어오고 어떤 데이터는 코드에 직접 쓰고 어떤 데이터는 서버로 온다.
어떤 데이터가 어디로 오는지 정도만 잘 구분하면 서버에 대해서 디테일하게 알 필요는 없다
서버와 클라이언트가 주고 받을 데이터를 잘 정해놓아야 한다 (아이템, 체력 같은거)
